Project Gradient Noise

Perlin-Style Noise

Project Overview

Gradient Noise generates seamless procedural textures from lattice gradients instead of raw random values. This demo renders an improved Perlin-style 3D noise slice into an offscreen floating-point texture, then reuses that texture for both a flat preview and a displaced surface view.

Tasks Completed

  • Added the Gradient Noise demo to the shared CS250 demo factory and CMake build.
  • Extended texture loading with RGBA8, RGBA32F, and R32F allocation formats.
  • Added framebuffer support for rendering generated noise into a repeat-wrapped R32F texture.
  • Implemented improved 3D gradient noise in GLSL with quintic fading and fixed gradient directions.
  • Added gradient, fractal sum, turbulence, marble, and wood pattern modes.
  • Built texture and surface shaders so the same generated noise can be inspected as a 2D texture or as displaced terrain.

Reflection

The biggest difference from value noise is that the random data now influences direction instead of direct color. That makes the transitions smoother and gives the higher-level patterns more natural motion when changing the Z slice or tiling scale.
